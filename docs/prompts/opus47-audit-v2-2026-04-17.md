## AUDIT SCOPE

### A. Interruption Handling (P0 — not yet implemented)

Current state: No watchdog. If GoClaw crashes or hangs, the n8n execution runs to timeout (600s) then silently reports `agent_error`. The browser session on Mac/Alpine remains open.

**Investigate and implement:**
1. How does GoClaw signal "still alive" during a long survey?
   - Check `22-heartbeat-system.md` — is heartbeat usable as a progress signal?
   - Check if GoClaw returns partial output or streams
2. What happens to PinchTab sessions when GoClaw dies? (Read `10-browser-pinchtab.md`)
3. Implement: after n8n timeout, call a cleanup endpoint or shell command to close orphaned browser sessions
4. Implement: n8n `onError` branch that sends a Telegram alert with `status=interrupted` before the execution dies silently

Specific change needed in n8n: add error branch from `Primary GoClaw Responses API` that closes orphaned sessions and sends notification.

### B. Survey-Specific Agent Prompt Quality (P1)

Current agent instructions (injected via n8n `primaryInstructions`) cover the happy path but have gaps:

**Investigate these specific scenarios and improve the prompts:**

1. **CAPTCHA**: What should the agent do? Options: (a) wait and retry, (b) report `needs_manual_review`, (c) use a CAPTCHA-solving approach if available. Check if PinchTab has any CAPTCHA tooling.

2. **Multi-step login / session expiry**: Survey portals (meinungsplatz.ch, surveyplaces.ch) sometimes redirect to a login page mid-survey. Current instructions don't handle this. Add: "If redirected to a login page, use the portal credentials from your workspace SOUL.md and re-authenticate before continuing."

3. **Matrix/grid questions**: Questions with a grid layout (rows × columns). Current: "one action at a time" — but matrix questions may need a different strategy. Specify: "For matrix questions, fill each row's answer separately before moving to next row."

4. **Consistency**: If the same question appears on multiple pages (validation trap), the answer must be identical. Add: "If a question repeats a previous question verbatim, give the exact same answer as before."

5. **Score calibration**: Currently "7-8 out of 10". Some panels screen out responses that are consistently 7-8 (detect bots). Consider: use varied scores (6-9) based on topic positivity. Update SOUL.md with a scoring strategy table.

6. **Language**: Swiss surveys arrive in FR, DE, IT. Current: agent presumably reads UI language and responds. Verify this is explicit. Add: "Answer in the language the survey uses. Do not switch languages mid-survey."

7. **"Je paie moi-même" coverage**: Current instruction mentions payment independence for French only. Add German: "Ich bezahle selbst" and Italian: "Pago io stesso".

**Deliverable:** Updated `primaryInstructions` and `fallbackInstructions` strings in the n8n `Analyze Email + Extract Link` node, plus updated `AGENTS.md` files in live workspaces.
